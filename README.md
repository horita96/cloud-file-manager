# 📁 クラウドファイルマネージャー (Cloud File Manager)

5月の就職活動に向けて開発を進めている、Java（バックエンド）とAWS（クラウドインフラ）を連携させた、堅牢なファイル管理システムです。

現在進行形で設計・開発を行っており、日々の学習の成果をこちらのコミット履歴にてリアルタイムで公開しています。

---

## 💡 概要・目的

- **概要**：CUIでJavaからファイルを選択して、AWS S3に保存するクラウドファイルマネージャの作成。
- **目的**：VBA開発と基本情報技術者の知識をベースに、Java（オブジェクト指向）とAWS（S3）を学習することによるモダン開発の理解。

---

## 💡 設計思想

本プロジェクトでは、実務のチーム開発や長期的なシステム保守を想定し、**「変更に強いオブジェクト指向設計（疎結合）」**を最大のテーマにしています。

### 1. インターフェースを用いた多態性（ポリモーフィズム）の実践
ファイルの保存処理を行うにあたり、メインの司令塔（`Main`）が直接特定の保存処理を呼び出すのではなく、共通の規格である `StorageService` インターフェースを経由する設計にしています。
これにより、最初に構築する「手元のPCに保存するパーツ（`LocalStorageService`）」から、後から実装する「AWSに保存するパーツ（`S3StorageService`）」へ切り替える際、
**メインの処理コードを1文字も書き換えることなく、プラグを差し替えるように機能を拡張できる設計**にしています。

### 2. セキュリティとコストを両立したインフラ構築
構築したAWS環境の知見を活かし、セキュリティ（`IAM / MFA`）とコスト（`S3`）を両立した設計にしています。

## 🗺️ 開発ロードマップ & 学習ステータス

ポートフォリオの完成、および実務で通用する堅牢なアプリケーション開発に向けて、以下の工程で進めています。

### 🟩 工程 0：インフラ基礎構築（期間：2025年5月18日） 🎉 完了
本番環境のベースとなるAWSアカウントの安全な利用環境と、開発PCからの接続基盤を構築しました。

- [x] **0-1. AWS CLIの導入と疎通確認**
  - **ゴール**: ローカル環境（Windows）にAWS CLIを導入し、コマンドラインからAWSリソースへ安全にアクセスできることを確認する。
- [x] **0-2. IAMユーザーの作成およびMFA（多要素認証）の設定**
  - **ゴール**: ルートアカウントの利用を避け、最小限の権限を与えた作業用IAMユーザーを作成。不正アクセスを防ぐためMFAを有効化し、セキュリティ基盤を確立する。


### 🛠️ 工程 1：オブジェクト指向とAPI活用術の習得（期間目安：3〜4週間） 🔄 現在進行中
本番アプリを設計・実装するための基礎体力を、Javaの標準的なオブジェクト指向マナーと主要API、例外処理を網羅しながら習得します。

- [x] **1-1. オブジェクト指向の基礎とインスタンス化**　
  - **ゴール**: クラスから `new` でインスタンス（実体）を生み出す意味を理解し、コンストラクタによる安全な初期データ設計を習得する。
- [x] **1-2. インタフェースと多態性の習得　✏️now!**
  - **ゴール**: 共通の看板（ルール）を作る「インタフェース」と、中身のクラスを自在に切り替える「多態性（ポリモーフィズム）」を理解する。
- [x] **1-3. カプセル化の徹底**
  - **ゴール**: データの安全性を高めるため、変数に `private` を付与し、`getter / setter` を通してのみアクセスさせる設計を徹底する。
- [x] **1-4. 主要API（静的メンバ・文字列操作・コレクション）の習得**
  - **ゴール**: `static` メソッド、文字列の切り出し（拡張子判定）、可変長配列（`ArrayList`）を使ったデータ一覧の扱い方を習得する。
- [x] **1-5. 例外処理の習得**
  - **ゴール**: 通信障害やファイル不在などのトラブルが発生しても、システムが強制終了せずに安全に生き残るための `try-catch` 構文をマスターする。


### 📐 工程 2：本番アプリの「骨組み」設計とローカル実装
AWSとの連携前に、PCのローカル環境を対象に、学んだオブジェクト指向設計を総動員してアプリケーションの頑丈な土台を構築します。

- [x] **2-1. プロジェクト構造とパッケージの作成**
  - **ゴール**: Javaの標準規約（すべて小文字・単数形）に準拠した、拡張性の高いパッケージ階層（独立型スタイル）を作成する。
- [ ] **2-2. 主要クラスの作成とCUIメニューの構築**
  - **ゴール**: `Main.java` で入力を受け付け、共通の看板である `StorageService` インターフェースを経由して、まずは `LocalStorageService` を呼び出す仕組み（疎結合設計）を作る。
- [ ] **2-3. 容量制限（バリデーション）の組み込み**
  - **ゴール**: 独自ユーティリティ `FileSizeLimiter.java` を実装。ファイルが指定サイズを超えていた場合、処理を安全に水際対策（シャットアウト）するロジックを組み込む。


### ☁️ 工程 3：AWS（S3）の環境構築と安全な鍵の設定
クラウド側の受け入れ態勢を整えるため、AWSマネジメントコンソール上でのインフラ構築と、セキュリティに配慮した環境設定を行います。

- [ ] **3-1. Amazon S3 バケットの作成**
  - **ゴール**: ファイルを保存するストレージ（S3バケット）を「完全非公開」の設定で堅牢に作成する。
- [ ] **3-2. IAMユーザーの作成と権限付与**
  - **ゴール**: 最小限の原則に基づき、JavaプログラムがS3を操作するためだけの専用権限（`AmazonS3FullAccess`）を持ったIAMユーザーを作成する。
- [ ] **3-3. 安全なアクセスキーの発行と環境変数への設定**
  - **ゴール**: 認証情報をソースコードにハードコーディングするリスクを避けるため、OSの環境変数にアクセスキーを安全に格納する。


### 🚀 工程 4：AWS SDK of 導入とJava実装（クラウド化）
すべてのパーツを統合し、ローカル駆動からAWS対応のクラウドストレージアプリケーションへと進化させます。

- [ ] **4-1. AWS SDK for Java の導入**
  - **ゴール**: ビルド管理ファイル（`pom.xml`等）に依存関係を記述し、JavaからAWSを操作するための公式ライブラリを自動導入する。
- [ ] **4-2. `S3StorageService.java` の実装**
  - **ゴール**: `StorageService` インターフェースの規約に従い、中身をAWS SDKのコードに置き換えたクラウド担当クラスを実装。環境変数から鍵を自動読込させて安全に通信を行う。
- [ ] **4-3. メイン処理との結合と堅牢な例外処理の肉付け**
  - **ゴール**: `Main.java` の呼び出し対象をAWS用へと切り替える。通信エラー等を想定した `try-catch` を徹底的に仕込み、現場仕様の頑丈なアプリに仕上げる。
