# 📁 クラウドファイルマネージャー (Cloud File Manager)

5月の就職活動に向けて開発を進めている、Java（バックエンド）とAWS（クラウドインフラ）を連携させた、セキュアなファイル管理システムです。

現在進行形で設計・開発を行っており、日々の学習の成果をこちらのコミット履歴にてリアルタイムで公開しています。

---

## 🚀 開発ロードマップと現在の進捗

基本情報技術者試験で得た体系的な知識をベースに、以下の4ステップで開発を進めています。

- [x] **フェーズ1：インフラ基礎構築**
  - AWS CLIの疎通確認、IAMユーザーおよびMFA（二段階認証）によるセキュリティ基盤の構築。
- [/] **フェーズ2：Javaによるオブジェクト指向設計（←今ココ）**
  - 複数クラスを連携させたアプリケーションの基本設計。
  *変化に強い「疎結合」な設計*を意識し、インターフェースを用いた部品化を進めています。
- [ ] **フェーズ3：AWS SDKを用いたS3連携機能の実装**
  - JavaプログラムからAWS APIを叩き、実際のクラウドストレージ（S3）へファイルを自動転送する処理の実装。
- [ ] **フェーズ4：ビジネスロジックの追加と検証**
  - ファイルのバリデーション（サイズチェック）や、アップロード時の拡張子自動変換（PNGからJPGへの自動変換など）の実装。

## 💡 こだわりの設計思想（面接官の皆様へ）

本プロジェクトでは、実務のチーム開発や長期的なシステム保守を想定し、**「変更に強いオブジェクト指向設計（疎結合）」**を最大のテーマにしています。

### 1. インターフェースを用いた多態性（ポリモーフィズム）の実践
ファイルの保存処理を行うにあたり、メインの司令塔（`Main`）が直接特定の保存処理を呼び出すのではなく、共通の規格である `FileStorage` インターフェースを経由する設計にしています。
これにより、最初に構築する「手元のPCに保存するパーツ（`LocalStorage`）」から、後から実装する「AWSに保存するパーツ（`S3Storage`）」へ切り替える際、**メインの処理コードを1文字も書き換えることなく、プラグを差し替えるように機能を拡張できる設計**にしています。

### 2. 前職の実績（自動化）とのつながり
前職ではVBAを用いて「手作業で3時間かかっていた業務を2分に短縮（約99%削減）」という業務効率化を達成しました。
その際に体感した「ITで課題を解決するおもしろさ」をベースに、より大規模でモダンなシステム開発に対応できるよう、本アプリの開発を通じて「Javaによるオブジェクト指向」と「AWSによるクラウドインフラ」の技術を自発的にキャッチアップしています。
